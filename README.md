## Instruções

1. Resolva cada desafio no [Beecrowd](https://judge.beecrowd.com) para validar sua solução

3. Faça `git add`, `git commit` e `git push` para enviar

## Critérios de Avaliação

### LTP3 (50%)
- Código funcional e lógica correta
- Uso adequado de strings, validações e condicionais
- Boas práticas de programação (nomes de variáveis, indentação)

### QP3 (50%)
- Código sem erros de sintaxe
- Arquivos organizados corretamente
- Entrega no prazo via Git
- Uso de validações (try/except, if/else)

## Como entregar

```bash
# Clone o repositório (apenas na primeira vez)
git clone <URL_DO_SEU_REPOSITORIO>
cd <NOME_DO_REPOSITORIO>

# Após editar os arquivos .py
git add .
git commit -m "resolvido desafio XXXX"
git push
```

## Nota Parcial

Cada desafio vale **20 pontos**. Você pode entregar parcialmente:
- 1 desafio = 20/100
- 3 desafios = 60/100
- 5 desafios = 100/100

A correção automática roda a cada push. Veja o resultado na aba **Actions** do seu repositório.
